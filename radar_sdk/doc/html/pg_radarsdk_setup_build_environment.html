RFS SDK Documentation
Infineon RFS SDK for evaluation and development of applications with Infineon XENSIV 60GHz radar sensors, PAS CO2 sensors, and Multigas sensors.
Building SDK from source code

The following compilers and platforms are officially supported:

  • Windows, x86-64 (Visual Studio 2017 or higher)
  • Windows, MinGW, x86-64 (GCC)
  • Linux, x86-64 (Ubuntu 22.04, GCC)
  • Linux, armhf (Raspbian Buster on Raspberry Pi, GCC)

Other combinations of platforms and compilers are not tested. The SDK may or may not compile and work with non-supported compilers and platforms.

Windows

On Windows the SDK can either be built using the free MinGW environment, using gcc within the Windows Subsystem for Linux (wsl) or using Microsoft Visual Studio.

MinGW

MinGW is a free and open source software development environment. It includes a port of the GNU compiler collection to Windows and allows to build libraries and programs for Microsoft Windows.

Setup build environment

You need to install the MinGW environment, cmake and ninja:

  • We recommend to download the MinGW distribution available at winlibs.com. Download the latest distribution for 64bit Windows (Architecture: x86_64) and extract the archive. In the following we assume that the distribution has been extracted to C:\mingw64\.
  • Download the latest version of cmake and install it. In the following we assume that cmake has been installed to C:\Program Files\CMake\ (this is the default install path).
  • Download the latest version of ninja and copy the executable ninja.exe into the directory C:\mingw64\bin\.
Compilation

To build the project using MinGW, please follow the following instructions:

  • Open a Windows command prompt (cmd) and change into the directory of the SDK release package.
  • Add the directory c:\mingw64\bin to the PATH: 
    $ SET PATH=c:\mingw64\bin;%PATH%
  • Create a new folder build and set it as working directory: 
    $ mkdir build/
$ cd build
  • Generate build files for Ninja using the cmake command: 
    $ "c:\Program Files\CMake\bin\cmake.exe" -G Ninja ..
  • Compile the source code using the ninja command: 
    $ ninja

The compiled executables and libraries have been created in the directory build\bin. The compiled libraries (.dll files) and executables (.exe files) depend on libraries of the MinGW environment. For this reason the MinGW distribution must be found, i.e., the MinGW distribution must be added to the PATH as described above.

Visual Studio

Download and install Visual Studio

The latest version of Visual Studio can be downloaded from the Microsoft Visual Studio Downloads page. Please follow the steps on the Install Visual Studio page to to install Visual Studio on your PC.

Create Visual Studio solution

To generate the Visual Studio Solution File (*.sln), you can use the CMake GUI. Please download the latest version of CMake and install it.

  • Run the CMake (cmake-gui) application.
  • Give the right directory paths:
    • Where is the source code: Path to the directory where the SDK source code has been extracted.
    • Where to build the binaries: This is the directory where the compiled artefacts are generated. We suggest to create a directory build in the source code directory.
  • Click Configure and Generate.

The Visual Studio solution file (*.sln) will be generated in the build directory.

Compilation

You can open the Visual Studio solution by clicking the button "Open Project" in the cmake tool. You can use Visual Studio to compile the solution or individual targets.

WSL (Windows Subsystem for Linux)

Download and install WSL

Before you can install a Linux distribution, WSL has to be enabled in one of the following ways:

Using the GUI for enabling Windows features

  • Open the Start Menu and search "Turn Windows features on or off"
  • Select Windows Subsystem for Linux
  • Click OK
  • Restart your computer when prompted

Using PowerShell

  • Open PowerShell as Administrator and run: 
    Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
  • Restart your computer when prompted

For more details please refer to the following links: Installation Instructions for WSL. We recommend to use Ubuntu 22.04 as distribution.

Linux

In the following, it is assumed that the operating system is Ubuntu 20.04 (x86-64), or Raspberry Pi (Raspbian GNU/Linux 10 Buster) is used.

Ubuntu 22.04 / Raspbian Buster

To build the project on Linux:

Prepare build environment

Before you start, make sure that all packages are up to date by executing the following command in a terminal: 

$ sudo apt update
$ sudo apt upgrade

Install cmake, ninja, and the gcc compiler: 

$ sudo apt install cmake ninja gcc build-essential
Compilation
  • Change to the directory of the source code.
  • Create a new directory build, and change to this directory: 
    $ mkdir build/
$ cd build
  • Generate the ninja files and build the project: 
    $ cmake ..
$ ninja

The built executables are located in the directory build/bin/.

Error while loading shared library

Please note that for some applications like app_segmentation, app_motion_angle, and app_advanced_motion_sensing it might be necessary to set the environment variable LD_LIBRARY_PATH such that the shared libraries are found. If you see an error message 

error while loading shared library

set the environment variable LD_LIBRARY_PATH to the directory where the compiled executables are located. Assuming you are in the directory of the executables, you can run app_segmentation using following commands: 

$ export LD_LIBRARY_PATH=. # . for working directory
$ ./app_segmentation       # run app_segmentation (or other application)

Python Wheels

To build the Python wheels for FMCW 60GHz and BGT60LTR11AIP manually, make sure that python is installed, and that pip, wheel, and setuptools are up to date: 

$ python -m pip install --upgrade pip wheel setuptools

The python wheels can be build using the target wheel-ifxradarsdk, which contains support for all IFX radar sensors: 

$ cmake --build . --target wheel-ifxradarsdk # Ifxradarsdk Python wrapper

The python wheels are then available in the directory pythons_wheels.

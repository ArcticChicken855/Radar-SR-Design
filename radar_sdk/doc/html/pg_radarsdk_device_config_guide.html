Device configuration guidelines

Radar device configuration parameters

FMCW parameters

Radar parameters are provided by the ifx_Avian_Config_t structure. An explanation of radar parameters is available at the introductory chapter.

Metrics

Alternatively, radar parameters can also be provided as "metrics" via the ifx_Avian_Metrics_t structure.

The SDK offers the possibility to convert metrics into FMCW parameters and vice-versa, through the ifx_avian_metrics_to_config() and ifx_avian_metrics_from_config() functions.

An explanation of metrics parameters and how these are converted into radar parameters is also available at the introductory chapter.

Note
For convenience, the example applications accept radar device configuration as either range/speed metrics or FMCW parameters.

Measurement Frame

Radar time-domain data contained in a measurement frame is obtained as a cube data structure.

The default interpretation of sequential data as a cube ifx_Cube_R_t (3-D array) is presented in the following diagram:

Ifx Cube Data Access

The figure above is an example of a ifx_Cube_R_t with 8 rows, 5 columns and 3 slices.

The measurement frame dimensions are translated to the ifx_Cube_R_t metrics as:

ifx_Cube_R_t Frame dimensions
Rows Number of Rx antennas for each active Tx antenna available in the device
Columns Number of chirps configured in a frame
Slices Number of samples per chirp

The data received by the radar device (each sequential sample is represented by a labelled cell) is subsequently arranged in an ifx_Cube_R_t as shown in the following diagram:

Radar Frame Storage in a Cube structure

In this example we assume 4 chirps/frame, 8 samples/chirp, 2 Tx and 4 Rx Antennas. Data corresponding to each transmit-receive antenna pair is considered data corresponding to one virtual antenna.

Virtual Rx antenna ordering

The virtual Rx antennas are ordered by [Tx][Rx] according to the number Tx and Rx antennas which are active.

For example, assuming all 4 Rx antennas and MIMO (2 Tx antennas) are activated:

Extract data from a single virtual antenna

Now, if we want to extract the data for a single Antenna, say virtual antenna 4, and place it in an ifx_Matrix_R_t we use the function ifx_cube_get_row_r with the required antenna number as parameter.

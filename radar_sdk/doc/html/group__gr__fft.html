Detailed Description

API for Fast Fourier Transform (FFT)

Supports one-dimensional FFT for real and complex input signals.

The Fourier transformation is an integral transform that composes a continuous function depending on space or time into functions depending on spatial or temporal frequency. The transform for a function \(f(x)\) is defined as

\[ \hat{f}(y) = \int_{-\infty}^\infty f(x) e^{- 2\pi i \, x\cdot y} \,\mathrm{d}x \]

where \(\hat{f}\) denotes the Fourier transform of \(f\). The transform is effectively a change of basis and expresses the original function \(f\) as a superposition of plane waves. It is a useful tool in signal processing for example to detect periodic components of a signal. Also note that other definitions of the Fourier transformation with different prefactors exist.

The Discrete Fourier Transform (DFT) corresponds to the Fourier transform for a discrete signal \(a_j\) \((j=0,1,\dots,N-1)\):

\[ \hat{a}_k = \sum_{j=0}^{N-1} e^{-2\pi i \cdot \frac{jk}{N}} \cdot a_j \quad (k=0,1,\dots,N-1) \]

For real input signals \(a_k = a_k^*\) ( \(a_k^*\) denotes the complex conjugate of \(a_k\)) the DFT contains only \(N/2+1\) independent values \(\hat{a}_0, \hat{a}_1, \dots, \hat{a}_{N/2}\) as \(\hat{a}_{N-k} = \hat{a}^*_k\). Note that the DC component \(\hat{a}_0\) and the component \(\hat{a}_{N/2}\) are both real.

The DFT can be computed effectively by means of the Fast Fourier Transform (FFT) algorithm with \(\mathcal{O}(N \log N)\) effort for an input signal of length \(N\).

More information can be found in standard textbooks or on Wikipedia:

  • https://en.wikipedia.org/wiki/Discrete_Fourier_transform
  • https://en.wikipedia.org/wiki/Fast_Fourier_transform

Performs FFT transform on a complex input.

Performs a discrete Fourier transform using the Fast Fourier Transform (FFT) algorithm. The FFT size \(N\) was specified in ifx_fft_create.

If the length of the input vector is smaller than \(N\), the input vector is internally zero padded. If the length of the input vector is larger than \(N\) only the first \(N\) elements are used and all other elements of the vector are ignored.

The FFT output has \(N\) complex elements and hence \(N\) elements are written to the vector output. The vector output must have at least length \(N\).

Parameters
[in]handleA handle to the FFT object
[in]inputComplex input vector
[out]outputComplex output vector (length of \(N\) or larger)

Performs FFT transform on a real input.

Performs a discrete Fourier transform on real input data using the Fast Fourier Transform (FFT) algorithm. The FFT size \(N\) was specified in ifx_fft_create.

If the length of the input vector is smaller than \(N\) the input vector is internally zero padded. If the length of the input vector is larger than \(N\) only the first \(N\) elements are used and all other elements of the vector are ignored.

If the length of the output vector is at least \(N\) the full \(N\) complex frequency samples are written to output.

If the length of the output vector is at least \(N/2+1\) only \(N/2+1\) complex frequency samples are written to output.

If the length of the output vector is \(N/2\) only \(N/2\) complex frequency samples are written to output.

The first element of output (index 0) contains the DC component and has imaginary part 0.

If the length of the output vector is smaller than \(N/2\) an error is set.

Parameters
[in]handleFFT object
[in]inputReal input samples
[out]outputComplex output vector

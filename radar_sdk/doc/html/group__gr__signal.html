Detailed Description

API for digital filtering operations.

Supports mathematical and other operations such as creation and destruction of filter objects, filtering operation and clearing filter state history.

Compute filter coefficients for Buttworth band-pass filter.

Compute coefficients of an Nth-order Butterworth band-pass filter. At the cuttoff frequencies the signal is damped by -3dB.

The filter coefficients can then be used to create a new filter using ifx_signal_filt_create_r. You can also use ifx_signal_filter_butterworth_create_r to create a Butterworth band-pass filter in one step.

The cutoff frequencies and the sampling frequency must fulfil: 0 < frequency_low_Hz < frequency_high_Hz < frequency_high_Hz/2

The vectors a and b must be allocated by the caller. The length of a and b must be order+1.

The current implementation becomes numerically unstable with higher orders. It is not recommended to use orders higher than 2.

See also: https://docs.scipy.org/doc/scipy/reference/generated/scipy.signal.butter.html.

Parameters
[in]orderorder of the Buttworth filter
[in]sampling_frequency_Hzsampling frequency in Hz
[in]frequency_low_Hzlower cutoff frequency in Hz
[in]frequency_high_Hzhigher cutoff frequency in Hz
[out]bnumerator polynomials of filter
[out]adenominator polynomials of filter

Cross-correlate two 1-dimensional arrays.

Cross-correlate the two 1-dimensional arrays x and and y. The result is written to z.

If mode is IFX_CORRELATE_FULL the output vector has the length \(\mathrm{len}(x)+\mathrm{len}(y)-1\) and is computed via

\[ z_k = \sum_{l=0}^{\mathrm{len}(x)-1} x_l \cdot y_{l + \mathrm{len}(y) - k - 1} \]

where \(k=0,1,\dots,\mathrm{len}(x)+\mathrm{len}(y)-2\), and \(y_m\) is zero if the index is outside of the valid range.

If mode is IFX_CORRELATE_SAME the output is a centered version of mode IFX_CORRELATE_FULL with dimension \(\mathrm{len}(x)\).

Parameters
[in]xFirst input vector
[in]ySecond input vector
[out]zVector of discrete linear cross-correlation of input1 and input2
[in]modeMode indicating size of output

◆ ifx_signal_filt_run_r()

IFX_DLL_PUBLIC void ifx_signal_filt_run_r ( ifx_Filter_R_t *  filter,
const ifx_Vector_R_t *  input,
ifx_Vector_R_t *  output 
)

IIR FIR Linear Filter Filters a data sequence, input, using a digital filter with M numerator coefficients vector b, and N denominator coefficients vector a.

The initial states (past samples of input and output) are set to zero. The filter is implemented as a direct II transposed structure. The formula for calculating the nth sample of output is expressed as

\[ \mathrm{output}_n = \frac{1}{a_0} \left(\sum_{j=0}^{M-1} b_j \cdot \mathrm{input}_{n-j} - \sum_{j=1}^{N-1}a_j \cdot \mathrm{output}_{n-j}\right) \]

input and output may point to the same vector.

Parameters
[in]filterFilter structure which includes the following pointers
  • b vector : the numerator coefficient vector
  • a vector : the denominator coefficient vector If a[0] is not 1, then the other coefficients are normalized by it's value.
  • state_b vector : FIR state from the last filtering operation
  • state_a vector : IIR state from the last filtering operation
[in]inputinput signal vector
[out]outputoutput vector(should be the same length as input)

Create a Butterworth filter.

This function creates a Butterworth filter. Following filter is created:

  • type=IFX_BUTTERWORTH_LOWPASS: low-pass filter with cutoff frequency cutoff_frequency1_Hz.
  • type=IFX_BUTTERWORTH_HIGHPASS: high-pass filter with cutoff frequency cutoff_frequency1_Hz.
  • type=IFX_BUTTERWORTH_BANDPASS: band-pass filter with cutoff frequencies cutoff_frequency1_Hz, cutoff_frequency2_Hz.

Note that unless type=IFX_BUTTERWORTH_BANDPASS the argument cutoff_frequency2_Hz is ignored.

The sampling frequency must be at least two-times the cutoff frequency.

The returned filter object can be destroyed after usage using the function ifx_signal_filt_destroy_r.

The current implementation becomes numerically unstable with higher orders. It is not recommended to use orders higher than 2.

[in]orderorder of Butterworth filter (must be positive).
[in]sampling_frequency_Hzsampling frequency in Hz (must be at least twice the cutoff frequency).
[in]cutoff_frequency1_Hzcutoff frequency in Hz.
[in]cutoff_frequency2_Hzcutoff frequency in Hz (only used for band-pass filter).
Returns

IFX_DLL_PUBLIC void ifx_signal_gaussianpulse_r ( const ifx_Vector_R_t *  t,

Generates a gaussian pulse vector.

Uses pulse configuration parameters \(b_w\) (pulse bandwidth) and \(f_c\)(center frequency) The output 'output' is the same size as 't'. The output is a sinusoid shaped by a gaussian function envelope. The \(\mathrm{bwr}\) or reference level at which fractional bandwidth is calculated is fixed at -6 (dB). The elements of the output are populated according to the following equation

\[ \mathrm{output}_n = e^{-\mathrm{a} {t_n}^{2}} \cdot cos(2 \pi f_c t_n) \]

where

\[ \mathrm{a} = \frac{-(\pi f_c b_w)^2}{4 \mathrm{ln}(10^{\frac{bwr}{20}})} \]

Parameters
[in]ttime vector defined by ifx_Vector_R_t
[in]fcpulse center frequency in Hz
[in]bwpulse bandwidth
[out]outputoutput vector defined by ifx_Vector_R_t contains gaussian pulse values. (should be equal in length to 't' input time vector.)

IFX_DLL_PUBLIC ifx_Hilbert_R_t* ifx_signal_hilbert_create_r ( uint32_t  hilbert_order,
uint32_t  signal_length 
)

Creates a Hilbert Filter object ifx_Filter_R_t.

The 'hilbert_order' parameter can be used to control how large a response we consider. The applied range of the filter reduces to \( -\mathrm{N} \le\ n\le \mathrm{N} \) where

\[ \mathrm{N} = \left(hilbert\_order \cdot 2\right)-1 \]

making the effective filter length

\[ Filter\_length = \left(hilbert\_order \cdot 4\right)-1 \]

The larger the order, the more accurate the output.

Parameters
[in]hilbert_orderunique unsigned non zero values of hilbert filter. Results in filter length as shown above.
[in]signal_lengthlength of signal, helpful in reducing hilbert filter length. 0 for variable.
Returns
Hilbert Filter object ifx_Filter_R_t

IFX_DLL_PUBLIC void ifx_signal_hilbert_run_c ( ifx_Hilbert_R_t *  hilbert_object,
const ifx_Vector_R_t *  input,
ifx_Vector_C_t *  output 
)

Computes an analytical signal using the hilbert transform.

The imaginary component of the output analytical signal is computed using a discrete Hilbert transform filter with a band pass (95%) frequency response. The coefficients of this filter approximate the time domain response of a filter imparting a \(\pm 90^{\circ} \) shift to each frequency component of the input. See also: https://en.wikipedia.org/wiki/Hilbert_transform#Discrete_Hilbert_transform.
The elements of the filter are populated according to the following equation

\[ \mathrm{h[n]} \overset{\Delta}{=} \left\{ \begin{matrix} 0, \quad for \;\mathrm{n}\; even \\ \frac{2}{\pi n}, \;for\;\mathrm{n}\; odd \end{matrix} \right. \]

This model has a range from \( -\infty \le\ n\le \infty \) and has to be limited. A Hamming window is applied to this section in order to reduce the side lobe artifacts of a rectangular window. The imaginary part of the analytical signal is computed through convolution and selection of the output with the zero phase as center. The real part of the analytical signal is the input signal itself.

Parameters
[in]hilbert_objectinput object defined by ifx_Hilbert_R_t
[in]inputinput signal vector defined by ifx_Vector_R_t
[out]outputoutput complex analytical signal vector defined by ifx_Vector_C_t

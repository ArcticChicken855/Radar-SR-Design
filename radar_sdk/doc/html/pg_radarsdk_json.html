RFS SDK Documentation
Infineon RFS SDK for evaluation and development of applications with Infineon XENSIV 60GHz radar sensors, PAS CO2 sensors, and Multigas sensors.
JSON Configuration

Configuration structure

The schema of the configuration (given in js-schema syntax) is 

{
    "?device": device_object,
    "?device_config": device_config_object,
    "?device_info": device_info_object,
    "?presence_sensing": presence_sensing_object,
    "?segmentation": segmentation_object,
    "?advanced_motion_sensing": advanced_motion_sensing_object,
    "?json_version": String
}

where a starting question-mark (?) marks a property as optional. A valid configuration may contain the properties device, device_config, device_info, presence_sensing, and segmentation. The property json_version is a string of the form x.y where x and y are numbers. If the property json_version is omitted, version 1.0 is assumed.

The schema of the objects device_object, device_config_object, device_info_object, presence_sensing_object, segmentation_object, and advanced_motion_sensing_object are described in the next sections.

Description of properties:

  • device [optional]: Description for which radar devices the configuration is applicable.
  • device_config [optional]: Description of the device configuration.
  • device_info [optional]: Additional information about the connected device.
  • presence_sensing [optional]: Configuration for the presence sensing application.
  • segmentation [optional]: Configuration for the segmentation application.
  • advanced_motion_sensing: Configuration for the advanced motion sensing algorithm (for BGT60LTR11AIP).
  • json_version [optional, default: "1.0"]: Version number of JSON configuration in the form x.y.

Schema of device

Note
Please note that the schema device is not supported as of now.

The schema of device is: 

{
    "types": Array.Of(String),
    "uuids": Array.Of(String)
}

Description of properties:

  • types [optional]: Array with types for which the configuration is applicable. Valid entries are "BGT60TR13C", "BGT60ATR24C", "BGT60UTR11AIP", "BGT60LTR11AIP".
  • uuids [optional]: Array with unique ids (uuids) of applicable radar sensors.

An application can read this section to determine to which sensor it may connect. The application must iterate over the list of uuids and check if a board with the same unique id is found, then connect to this board. If no such board was found, the application should iterate over types and connect to the radar sensor, if such a sensor type is found. If no sensor with a unique id contained in uuids is found and if no sensor with a type contained in types is found, the application must not connect to any sensor.

The same uuid must not be an entry of types in different device sections. If the section device is missing, the corresponding configuration is treated as a default configuration if no other configuration is found for the given radar device.

Schema of device_info

The schema of device_info is: 

{
    "baseboard_name": String,
    "device_name": String,
    "firmware_version": String
    "num_rx_antennas": Number,
    "num_tx_antennas": Number,
}

Description of properties:

  • baseboard_name: Human-readable name of the baseboard.
  • device_name: Human-readable name of the device.
  • firmware_version: Firmware version.
  • num_rx_antennas: Number of RX antennas of sensor.
  • num_tx_antennas: Number of TX antennas of sensor.

Schema of device_config

The schema of device_config is: 

{
    "fmcw_scene": fmcw_scene_object,
    "fmcw_single_shape": chirp_timings_object,
    "doppler_ltr11": doppler_ltr11_object
}

The schemas of fmcw_scene and fmcw_single_shape are described in the following sections. The device_config object must contain either fmcw_scene or fmcw_single_shape or `doppler_ltr11". Setting two objects is not allowed.

Description of properties:

  • fmcw_scene: Configuration in terms of easy to understand high-level parameters (Avian sensors).
  • fmcw_single_shape: Configuration of one single shape (Avian sensors).
  • doppler_ltr11: Doppler configuration (BGT60LTR11AIP).

Schema of fmcw_scene

The fmcw_scene schema is: 

{
    "rx_antennas": Array.Of(Numbers),
    "tx_antennas": Array.Of(Numbers),
    "tx_power_level": Number,
    "if_gain_dB": Number,
    "range_resolution_m": Number,
    "max_range_m": Number,
    "max_speed_m_s": Number,
    "speed_resolution_m_s": Number,
    "frame_repetition_time_s": Number,
    "?sample_rate_Hz": Number,
    "?center_frequency_Hz": Number
}

Description of properties:

  • rx_antennas [required]: Array of integers with active RX antennas. E.g., to activate RX antennas 1 and 3: [1,3]
  • tx_antennas: Array of integers with active TX antennas. E.g., to activate TX antennas 1 and 2: [1,2]
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values are [0-31].
  • if_gain_dB [required]: Amplification factor in decibel that is applied to the IF signal coming from the RF mixer before it is fed into the analog-to-digital converter (ADC). Allowed values are [18-31].
  • range_resolution_m [required]: Defines the resolution of the radar device. That means the ability to resolve two objects with similar distance to the radar. The value is given in units of meters. The frequency difference of two adjacent bins in the range transform corresponds to range_resolution_m.
  • max_range_m [required]: Maximum distance to the radar that can be detected in units of meters. The bins of the range transform correspond to range values between 0 and max_range_m.
  • max_speed_m_s [required]: Maximum measurable radial velocity of a target from the radar sensor in units of meters per second.
  • speed_resolution_m_s [required]: Desired speed resolution that one wants to achieve with micro-Doppler measurement. That means what velocity difference one can detect for two targets at the same distance from the radar. The value is in units of meters per second. The frequency difference of two adjacent bins of the Doppler transform correspond to speed_resolution_m_s.
  • frame_repetition_time_s [required]: Time between two consecutive frames in seconds and the inverse of the frame rate. Also known as frame period or frame time.
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.
  • center_frequency_Hz [optional]: Center frequency of the chirp in Hz. The center frequency is defined as the mid of the start frequency and the end frequency of the chirp. If omitted the default value for the center frequency from the radar device is used.

The start and end frequency of the FMCW ramp are computed from range_resolution_m and center_frequency_Hz. The number of ADC samples per chirp is computed from max_range_m. The number of chirps per frame is computed from speed_resolution_m_s. The pulse repetition time (PRT) is computed from max_speed_m_s.

Schema of fmcw_single_shape

The fmcw_single_shape schema is: 

{
    "rx_antennas": Array.Of(Numbers),
    "tx_antennas": Array.Of(Numbers),
    "tx_power_level": Number,
    "if_gain_dB": Number,
    "start_frequency_Hz": Number,
    "end_frequency_Hz": Number,
    "num_chirps_per_frame": Number,
    "num_samples_per_chirp": Number,
    "chirp_repetition_time_s": Number,
    "frame_repetition_time_s": Number,
    "?hp_cutoff_Hz": Number,
    "?aaf_cutoff_Hz": Number,
    "?mimo_mode": String,
    "?sample_rate_Hz": Number
}

Description of properties:

  • rx_antennas [required]: Array of integers with activated RX antennas. E.g., to activate RX antennas 1 and 3: [1,3]
  • tx_antennas [required]: Array of integers with activated TX antennas. E.g., to activate TX antennas 1 and 2: [1,2]
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values are [0-31].
  • if_gain_dB [required]: Amplification factor in decibel that is applied to the IF signal coming from the RF mixer before it is fed into the analog-to-digital converter (ADC). Allowed values are [18-31].
  • start_frequency_Hz [required]: Start frequency of the FMCW chirp in Hz.
  • end_frequency_Hz [required]: End frequency of the FMCW chirp in Hz.
  • num_chirps_per_frame [required]: Number of chirps in a single frame.
  • num_samples_per_chirp [required]: Number of samples in a single chirp (data frame).
  • chirp_repetition_time_s [required]: Time between the start of two consecutive chirps in a frame in seconds. Also known as chirp-to-chirp time or pulse repetition time.
  • frame_repetition_time_s [required]: Time between two consecutive frame in seconds and the inverse of the frame rate. Also known as frame period or frame time.
  • hp_cutoff_Hz [optional, default: 80000]: Cutoff frequency of the high pass filter in Hz.
  • aaf_cutoff_Hz [optional, default: 500000]: Cutoff frequency of the anti aliasing filter in Hz.
  • mimo_mode [optional, default: "off"]: Configuration of MIMO mode. Possible values are:
    • "off": MIMO is disabled.
    • "tdm": Time-division multiplexing. This value is currently only allowed with ATR24C with both transmitting antennas enabled.
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.

Schema of fmcw_device_config

The fmcw_device_config schema is: 

{
    "sequence": [
    {
      "type": "loop",
      "sequence": [
        {
          "type": "chirp",
          "start_frequency_Hz": Number,
          "end_frequency_Hz": Number,
          "hp_cutoff_Hz": Number,
          "if_gain_dB": Number,
          "lp_cutoff_Hz": Number,
          "num_samples_per_chirp": Number,
          "rx_mask": Number
          "sample_rate_Hz": Number,
          "tx_mask": Number,
          "tx_power_level": Number
        },
        {
          "type": "delay",
          "time_s": Number,
        },
        {
          "type": "chirp",
          "start_frequency_Hz": Number,
          "end_frequency_Hz": Number,
          "hp_cutoff_Hz": Number,
          "if_gain_dB": Number,
          "lp_cutoff_Hz": Number,
          "num_samples_per_chirp": Number,
          "rx_mask": Number
          "sample_rate_Hz": Number,
          "tx_mask": Number,
          "tx_power_level": Number
        }
      ],
      "num_repetitions": Number,
      "repetition_time_s": Number
    }
  ]
}

Description of properties:

  • sequence [required]: This structure is the base of all acquisition sequence elements.
  • type [required]: Acquisition sequence is build from ifx_Fmcw_Sequence_Loop_t, ifx_Fmcw_Sequence_Chirp_t and ifx_Fmcw_Sequence_Delay_t.
  • start_frequency_Hz [required]: Start frequency of the FMCW chirp in Hz.
  • end_frequency_Hz [required]: End frequency of the FMCW chirp in Hz.
  • hp_cutoff_Hz [optional, if not set then default is assigned based on Device]: Cutoff frequency of the high pass filter in Hz.
  • if_gain_dB [required]: Amplification factor in decibel that is applied to the IF signal coming from the RF mixer before it is fed into the analog-to-digital converter (ADC). Allowed values for Avian device are [18-60].
  • lp_cutoff_Hz [optional, if not set then default is assigned based on Device]: Cutoff frequency of the low pass (anti aliasing) filter in Hz.
  • num_samples_per_chirp [required]: Number of samples in a single chirp (data frame).
  • rx_mask [required]: Bit mask of activated RX antennas
  • sample_rate_Hz [optional, default: 1MHz]: Sample rate of the analog-to-digital converter (ADC) in Hz.
  • tx_mask [required]: Bit mask of activated TX antennas
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values for Avian device are [0-31].
  • time_s [required]: The delay period in seconds.This is the time period in seconds inserted between the previous and the following loop element
  • num_repetitions [required]: The number of loop repetitions. A value of 1 means the sub-sequence is executed only once (so actually not repeated). This value may be 0 to indicate an infinite number of executions. The latter makes only sense if the loop is the last element in the sequence.
  • repetition_time_s [required]: The reciprocal repetition rate. This is the time period in seconds that elapses between the beginnings of two consecutive repetitions of the looped sub-sequence. A value of 0 means "as fast as possible". In that case, the duration of the looped sub-sequence defines the repetition rate.

Schema of doppler_ltr11

The doppler_ltr11 schema is: 

{
    "mode": String,
    "num_of_samples": Number,
    "pulse_repetition_time_us": Number,
    "pulse_width_us": Number,
    "rf_frequency_Hz": Number,
    "if_gain_dB": Number,
    "tx_power_level": Number
}

Description of properties:

  • mode [required]: Either "pulsed" for pulsed mode or "cw" for continuous wave mode.
  • num_of_samples [required]: Number of samples.
  • pulse_repetition_time_us [required]: Pulse repetition time in microseconds. Allowed values are 250, 500, 1000, and 2000.
  • pulse_width_us [required]: Pulse width in microseconds. Allowed values are 3, 4, 5, and 10.
  • rf_frequency_Hz [required]: Operational RF center frequency in Hz.
  • if_gain_dB [required]: IF gain in decibel. Valid values are 10, 15, 20, 25, 30, 35, 40, 45, and 50.
  • tx_power_level [required]: The value determines the power of the emitted signal. Higher values correspond to higher power settings. The values do not directly correspond to a physical quantity. Allowed values are [0-7].

Segmentation configuration

The schema for segmentation is given as: 

{
    "orientation": String
}

Description of properties:

  • orientation [optional]: Orientation may be either "landscape" or "portrait". If not specified the default value is landscape.

Presence Sensing

The schema for presence_sensing is: 

{
    "min_detection_range_m": Number,
    "max_detection_range_m": Number,
    "range_hysteresis_percentage": Number,
    "absence_confirm_count": Number,
    "presence_confirm_count": Number
}

Description:

  • min_detection_range_m [required]: Targets below this distance are ignored by the presence sensing algorithm. The value is in units of meters.
  • max_detection_range_m [required]: Targets above this distance are ignored by the presence sensing algorithm. The value is in units of meters.
  • range_hysteresis_percentage [required]: Hysteresis for max_detection_range_m while changing state from presence to absence and vice versa. The value are [0-100].
  • absence_confirm_count [required]: Number of consecutive absence detections in consecutive frames for making a definitive presence detection.
  • presence_confirm_count [required]: Number of consecutive presence detections in consecutive frames for making a definitive presence detection.

Advanced Motion Sensing Algorithm

The schema for advanced_motion_sensing is given as: 

{
    "interference_mitigation": Boolean,
    "threshold": Number
}

Description of properties:

  • interference_mitigation [required]: When true the interference mitigation is enabled.
  • threshold: Threshold for algorithm. Valid values are in the range [10, 100].

Example configurations

Using fmcw_scene

Example how to configure a radar chip using the scene configuration: Antennas RX1, RX2, RX3, and TX1 are activated, the range resolution is set to 15cm, the maximum range to 10m, the speed resolution to 0.2m/s, and the maximum speed to 5m/s. The frame repetition time (or frame period) is 0.1s and corresponds to a frame rate of 1/0.1s = 10Hz. The ADC sample rate has the default value of 1MHz. 

{
    "device_config": {
        "fmcw_scene": {
            "rx_antennas": [1, 2, 3],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "range_resolution_m": 0.15,
            "max_range_m": 10,
            "max_speed_m_s": 5,
            "speed_resolution_m_s": 0.2,
            "frame_repetition_time_s": 0.1
        }
    }
}

Using fmcw_single_shape

Example how to configure a radar chip using the single shape configuration: MIMO mode is deactivated, the antennas RX1 and TX1 are activated, the chirp frequencies are set to 59GHz-61GHz. During each chirp 32 samples are taken and a frame consists of 128 chirps. The pulse repetition time is 0.001s (1ms) and the frame repetition time (or frame period) is 0.1s which corresponds to a frame rate of 10Hz. The ADC sample rate has the default value of 1MHz. 

{
    "device_config": {
        "fmcw_single_shape": {
            "rx_antennas": [1],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "start_frequency_Hz": 59e9,
            "end_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    }
}

The following example is similar to the last one but with MIMO enabled. The MIMO mode is set to "tdm" (time-domain-multiplexing) and the TX antennas TX1 and TX2 are activated. This example only works with ATR24C. 

{
    "device_config": {
        "fmcw_single_shape": {
            "mimo_mode": "tdm",
            "rx_antennas": [1],
            "tx_antennas": [1,2],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "start_frequency_Hz": 59e9,
            "end_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    }
}

{
    "device_config": {
        "fmcw_single_shape": {
            "sample_rate_Hz": 1e6,
            "mimo_mode": "off",
            "rx_antennas": [1],
            "tx_antennas": [1],
            "tx_power_level": 30,
            "if_gain_dB": 30,
            "start_frequency_Hz": 59e9,
            "end_frequency_Hz": 61e9,
            "num_chirps_per_frame": 128,
            "num_samples_per_chirp": 32,
            "chirp_repetition_time_s": 0.001,
            "frame_repetition_time_s": 0.1
        }
    },
    "segmentation": {
        "orientation": "landscape",
    }
}

Configuration for multiple devices

Note: This is not supported yet.

This example shows how to provide different configurations for different radar devices. The configurations are given as an array and the section device specifies for which types or uuids the configuration is valid. If the section device is missing, this configuration is treated as the default configuration in case no other matching configuration is found.

[ { "device": { "uuids": ["01234567-89ab-cdef-0123-456789abcdef"] }, "device_config": { "fmcw_scene": { "rx_antennas": [1, 2, 3], "tx_antennas": [1], "tx_power_level": 30, "if_gain_dB": 30, "range_resolution_m": 0.15, "max_range_m": 10, "max_speed_m_s": 5, "speed_resolution_m_s": 0.2, "frame_repetition_time_s": 0.1 } } }, { "device": { "types": [ "BGT60TR13" ], "uuids": [ "00034567-89ab-cdef-0123-456789abcdef", "01134567-89ab-cdef-0123-456789abcdef" ] }, "device_config": { "fmcw_scene": { "rx_antennas": [1, 2, 3], "tx_antennas": [1], "tx_power_level": 30, "if_gain_dB": 30, "range_resolution_m": 0.15, "max_range_m": 10, "max_speed_m_s": 5, "speed_resolution_m_s": 0.2, "frame_repetition_time_s": 0.1 } } } ]
